Aprenentatge supervisat¶

  • Prover a un algorisme d'aprenentatge (learning algorithm) dades d'entrenament de les quals pugui aprendre.
  • El training data est format per una srie d'atributs (features), i una columna amb la resposta correcte, la qual anomenem target (o label)
  • Durant la fase de training el learning algorithm troba patrons sobre el training data que "mapejen" els features al target (resposta que volem predir). Retorna un model que captura aquests patrons.
  • Per ltim podem fer servir aquest model per fer prediccions amb dades que no sabem el target.

"La prueba del algodón": Cross-Validation¶

Per saber si un model generalitza, s'ha de probar amb dades noves (out-of-sample).

Tcnica Cross-Validation :

  1. Separar el dataset en 2 conjunts (p.ex. 70%/30% de les dades)
  2. Entrenarem el nostre model amb el primer conjunt (training set)
  3. Aplicarem el model al segon conjunt (validation set)
  4. Mesurarem el performance del nostre model amb qualsevol mtrica d'evaluaci

Kaggle.com¶

Competicions de datascience amb regles senzilles:

  • Cada competici t 2 datasets pblics: training (labeled) i test
  • Els participants han de pujar un fitxer .CSV amb els camps ID (del test) i TARGET (la seva predicci). Aquest upload se'n diu submission
  • Sobre un percentatge d'aquestes prediccions, s'evaluen l'accuracy mitjanant la mtrica de la competici, i el submission apareix en el public leaderboard
  • Quan acaba el termini de la competici, s'evaluen tots els submissions amb la resta de dades del test set, i es crea un private leaderboard que determina els guanyadors finals.
